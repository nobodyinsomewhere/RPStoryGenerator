# RP Story Generator

> 一个纯前端、开箱即用、可直接部署的 RP 剧情触发与开场生成工具。  
> 支持角色卡导入（JSON / PNG）、AI 深度生成、本地保存、收藏、复制、导出，以及多主题玻璃风界面。

---

## 简介

RP Story Generator 是一个面向 RP / 酒馆 / 角色互动场景设计的轻量级前端工具。

它可以帮助你根据角色设定、关系类型、场景、剧情方向与额外需求，快速生成：

- 一句话钩子
- 场景摘要
- 推进节点
- 开场白

既可以使用模板快速生成，也可以接入 OpenAI 兼容 API 进行更自然、更贴近角色风格的深度生成。

本项目为纯前端实现，不依赖后端数据库，也不要求复杂环境配置。  
你可以本地直接打开使用，也可以部署到 Netlify、Vercel、Cloudflare Pages 等静态托管平台。

## 核心亮点

### 开箱即用
- 纯前端实现
- 无需后端
- 无需数据库
- 无需复杂环境配置
- 直接打开即可使用，也可轻松部署到线上

### 剧情生成更聚焦 RP 场景

- 默认数据保存在当前浏览器本地，无需注册登录

## 角色卡导入说明

本项目支持导入：

- JSON 角色卡
- PNG 角色卡

导入后会自动尝试提取以下信息：

- 角色名
- 描述
- 人格
- 场景
- 设定摘要
- 创作者备注（如存在）

### 关于 PNG 导入
PNG 角色卡的读取依赖图片中是否嵌入了可识别的角色卡数据。  
如果 PNG 本身不包含对应角色数据，工具将无法自动提取内容。

---

## 数据存储说明

本项目默认**不会将用户数据保存到服务器**。

用户输入内容、收藏结果等主要保存在：

- 浏览器本地存储（`localStorage`）
- 用户自己导出的 TXT / JSON 文件

这种方式的优点是：

- 更轻量
- 更易部署
- 更偏本地优先
- 不需要后端维护
- 对用户隐私更友好

同时也建议用户定期导出内容作为备份，以避免清理浏览器缓存后导致本地数据丢失。

## AI 配置说明

本项目支持 **OpenAI 兼容接口**。

用户可以在页面中填写：

- Base URL
- API Key
- Model

然后切换到 **AI 深度生成** 模式，让模型根据角色设定与剧情需求生成更自然的内容。

AI 输出主要用于补全或优化以下模块：

- 一句话钩子
- 场景摘要
- 推进节点
- 开场白

## 注意事项

- 默认不包含后端、数据库和账号系统
- 如果你需要云端存储、账号登录、多人同步等功能，需要额外接入后端服务

- AI 模式依赖用户自行配置 OpenAI 兼容接口
